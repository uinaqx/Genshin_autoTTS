# Genshin_autoTTS 快速开始

## 1. 启动

- 安装版：从开始菜单或桌面快捷方式启动 `Genshin_autoTTS`。
- 便携版：解压整个 ZIP 后运行 `Genshin_autoTTS.exe`，不要只复制单个 EXE。

首次启动会在 `%LOCALAPPDATA%\GenshinAutoTTS` 创建配置和缓存目录。

## 2. 设置游戏

1. 将游戏设为“窗口化”或“无边框窗口”。
2. 打开一段无配音对话并停留在完整字幕画面。
3. 在软件中先点击“全屏 16:9 预设”；如果识别不准，再分别框选角色名与字幕区域。
4. 点击“截屏识别测试”，确认日志中能看到角色和完整台词。

软件只读取屏幕像素，不注入进程、不读取游戏内存，也不修改游戏文件。

## 3. 配置云端语音

1. 在“云端语音服务”中选择“火山引擎”或“阿里云”。
2. 点击“配置 API”：
   - 火山引擎填写豆包语音控制台中的 `App ID` 和 `Access Token`。
   - 阿里云填写智能语音交互项目的 `AppKey` 和 `Access Token`。
3. 凭据会通过 Windows DPAPI 为当前用户加密，不会写入普通配置文件。
4. 输入测试角色和台词，点击“测试语音”。日志显示平台名和 `mp3` 即表示接入成功。

同一角色首次出现时会根据本地提示和稳定哈希分配音色，映射保存在
`%LOCALAPPDATA%\GenshinAutoTTS\speaker_profiles.json`，后续自动复用。

## 4. 开始运行

点击“开始自动配音”，切回游戏。程序会等待字幕稳定，在本地根据角色名选择固定音色，再将台词和音色 ID 发送给所选云平台。返回的音频会写入本地缓存并播放；相同角色和台词再次出现时不重复请求。

## 5. 常见问题

### OCR 无法识别

- 确认画面缩放与分辨率没有在框选后改变。
- 缩小框选范围，避开头像、任务提示和交互按钮。
- 使用“本地真实截图批量诊断”检查静态截图。

### 有识别结果但没有声音

- 先点击“测试语音”，确认 API 凭据、平台额度和扬声器正常。
- 火山引擎检查应用是否已开通语音合成、音色是否已授权。
- 阿里云检查 AppKey 与 Token 是否属于同一账号，以及 Token 是否已过期。
- 查看日志中的平台状态码、鉴权失败、并发超限或余额提示。

### 是否会上传截图或角色名

不会。截图、OCR 和角色到音色的映射都在本机完成。云端只收到稳定后的台词、音色 ID 以及语速、音量等必要合成参数。

### 仍想使用真人录音包

选择“真人录音包（兼容）”，再选择合法录音包的 `manifest.json`。该模式不请求云端合成；缺少对应录音时会拒绝播放。

### Windows 显示“未知发布者”

当前社区版本未购买代码签名证书。请只从本仓库的 GitHub Release 下载，并用同一 Release 中的 `SHA256SUMS.txt` 核对文件。不要从第三方网盘下载或关闭系统安全功能。
